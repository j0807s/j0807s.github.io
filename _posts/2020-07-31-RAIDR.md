* #### Problem : Refresh operations interfere memory accesses and waste energy
* #### Observation : Energy and performance overhead caused by refresh operations rised when the device integrity increased 
* #### Idea: Applied different refresh rate for strong DRAM cells (idea from [RAIDR(ISCA'2012)]({{ site.url }}/img/RAIDR.pdf))
* #### Implementation : Bloomfilter to store the information of weak rows in memory controller, DDR4-2133 configuration file
* #### Evaluation : Used DRAMSim2 and gem5 as simulators, memory intensive benchmarks from SPEC CPU2006 
* #### Contribution : Came up with the result that RAIDR averaged 12% performance improvement over auto-refresh scheme in DDR4
